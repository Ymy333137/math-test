# 长期看板与难度调节（dashboard）

## 1. 趋势统计
按单元（unit）独立统计：

- 掌握度趋势：最近 5 次 mastery 的均值、最高值、最低值。
- 难度变化趋势：最近 5 次 difficulty_level 的变化方向。
- 最近错误统计：最近 5 次 error_tags 的频率。
- 当前弱点：高频错误标签 + 低 mastery 题型特征。

## 2. 难度自动调节规则
难度等级：`Level 1` / `Level 2` / `Level 3`。

对同一单元，读取最近两次记录的表现等级（来自 analyzer 的“优秀/合格/不佳”）：

- 若最近两次均为“优秀” -> 下一题难度 +1
- 若最近两次均为“不佳” -> 下一题难度 -1
- 其他情况 -> 难度保持不变

边界限制：最低 `Level 1`，最高 `Level 3`。

## 3. 默认起始策略
若某单元暂无历史记录，则起始难度为 `Level 1`。

## 4. 输出建议模板
每次评估后输出：

1. 掌握度趋势（上升/持平/下降）
2. 难度变化趋势（稳定/上调中/下调中）
3. 最近错误统计（按频次）
4. 当前弱点（1~3 条）
5. 下一题难度预期（Level X）

## 5. 多题模式规则
- 当用户明确提出“多题模式”时，默认每轮出 3 题（同一单元、同一难度）。
- 每题独立评分并写入 `records`，每题都保留 `question_id` 与完整评估字段。
- 轮次总结额外给出：
  - 本轮 3 题平均 mastery
  - 本轮最高频错误标签
  - 下一轮难度预期（仍按“最近两次表现”规则，不因多题模式改变）
- 用户可指定题量（如“2题模式/4题模式”），未指定则按 3 题执行。


## 6. 单元完成与补测策略
- 单元是否“完成”，不只看分数，也看考纲覆盖度。
- 若用户要求“本单元一次性做完”，应优先从尚未覆盖知识点中组卷，题量不少于 3 题。
- 补测轮结束后输出：
  1. 已覆盖知识点
  2. 未覆盖知识点（若有）
  3. 是否建议进入下一单元


## 7. 语言与展示规范
- 面向用户展示的所有内容（题目、评语、趋势分析、学习建议）统一使用中文。
- PR 标题与 PR 正文统一使用中文。
- 若出现英文术语，可在括号中保留英文缩写，但主体表述必须为中文。


## 8. 快速扫描模式（Learning Rate 优先）
- 当用户提出“快速扫描/全面扫描”时，允许单轮出题量提升至 10 题或更多。
- 题目组织方式：按知识点由浅入深递进（基础计算 → 方法应用 → 综合判断）。
- 扫描模式下仍逐题评分并写入 `records`，但可在同一轮统一回收答案后一次性分析。
- 扫描目标：尽快定位薄弱知识点，而非控制单轮学习负荷。

# 长期看板与难度调节（dashboard）

## 1. 趋势统计
按单元（unit）独立统计：

- 掌握度趋势：最近 5 次 mastery 的均值、最高值、最低值。
- 难度变化趋势：最近 5 次 difficulty_level 的变化方向。
- 最近错误统计：最近 5 次 error_tags 的频率。
- 当前弱点：高频错误标签 + 低 mastery 题型特征。

## 2. 难度自动调节规则
难度等级：`Level 1` / `Level 2` / `Level 3`。

对同一单元，读取最近两次记录的表现等级（来自 analyzer 的“优秀/合格/不佳”）：

- 若最近两次均为“优秀” -> 下一题难度 +1
- 若最近两次均为“不佳” -> 下一题难度 -1
- 其他情况 -> 难度保持不变

边界限制：最低 `Level 1`，最高 `Level 3`。

## 3. 默认起始策略
若某单元暂无历史记录，则起始难度为 `Level 1`。

## 4. 输出建议模板
每次评估后输出：

1. 掌握度趋势（上升/持平/下降）
2. 难度变化趋势（稳定/上调中/下调中）
3. 最近错误统计（按频次）
4. 当前弱点（1~3 条）
5. 下一题难度预期（Level X）

## 5. 多题模式规则
- 当用户明确提出“多题模式”时，默认每轮出 3 题（同一单元、同一难度）。
- 每题独立评分并写入 `records`，每题都保留 `question_id` 与完整评估字段。
- 轮次总结额外给出：
  - 本轮 3 题平均 mastery
  - 本轮最高频错误标签
  - 下一轮难度预期（仍按“最近两次表现”规则，不因多题模式改变）
- 用户可指定题量（如“2题模式/4题模式”），未指定则按 3 题执行。


## 6. 单元完成与补测策略
- 单元是否“完成”，不只看分数，也看考纲覆盖度。
- 若用户要求“本单元一次性做完”，应优先从尚未覆盖知识点中组卷，题量不少于 3 题。
- 补测轮结束后输出：
  1. 已覆盖知识点
  2. 未覆盖知识点（若有）
  3. 是否建议进入下一单元


## 7. 语言与展示规范
- 面向用户展示的所有内容（题目、评语、趋势分析、学习建议）统一使用中文。
- PR 标题与 PR 正文统一使用中文。
- 若出现英文术语，可在括号中保留英文缩写，但主体表述必须为中文。


## 8. 快速扫描模式（Learning Rate 优先）
- 当用户提出“快速扫描/全面扫描”时，允许单轮出题量提升至 10 题或更多。
- 题目组织方式：按知识点由浅入深递进（基础计算 → 方法应用 → 综合判断）。
- 扫描模式下仍逐题评分并写入 `records`，但可在同一轮统一回收答案后一次性分析。
- 扫描目标：尽快定位薄弱知识点，而非控制单轮学习负荷。


## 9. 题号连续编号规则
- 全项目题号使用统一连续编号，不按单元重置。
- `question_id` 推荐格式：`Q0001`、`Q0002` ……，便于检索与统计总题量。
- 新题编号取 `records` 中最大题号 + 1。


## 10. 评测出题提示约束
- 为保证评测有效性，题目描述中不主动提示解法（如“可用洛必达/泰勒”）。
- 若用户作答后请求讲评，再在评语阶段给出可选方法与对比。

## 11. 题册与自定义题号规则
- 当用户使用外部题册训练时，允许采用自定义 `question_id`，例如 `800-1`、`800-2`。
- 此类记录建议增加：
  - `source_label`：如 `题册800`
  - `assessment_basis`：如 `self_report`
- 若题册训练不希望干扰正式单元难度，可单独建立对应 `unit`（如 `workbook_800`）保存历史与难度。
- 题册评估结束后，除常规趋势外，还应额外输出：
  1. 总题量与正确题量
  2. 高频错因
  3. 需要回炉的知识点

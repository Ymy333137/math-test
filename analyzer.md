# 数学训练评估标准（analyzer）

## 1. 评分维度
每次作答按照以下维度评估：

- 正确度（0-1）：结论与关键中间结果是否正确。
- 思维链质量（0-1）：推理是否完整、逻辑是否连贯、是否体现方法选择依据。
- 缺失步骤（数组）：应有但未给出的关键步骤列表。
- 易错点分类（数组）：按错误类型标注，例如概念误用、运算错误、定理适用条件遗漏、分类讨论不全。

## 2. 掌握度计算
掌握度（mastery）取值区间为 `[0, 1]`，默认按加权计算：

`mastery = 0.6 * 正确度 + 0.4 * 思维链质量`

等价字段写法：`mastery = 0.6 * correctness + 0.4 * thinking_quality`。

若存在致命错误（例如核心结论方向性错误且无法由后续步骤弥补），可在计算后再下调 `0.1 ~ 0.3`，最低不小于 `0`。

## 3. 表现等级
- 优秀：`mastery >= 0.8`
- 合格：`0.6 <= mastery < 0.8`
- 不佳：`mastery < 0.6`

## 4. 难度调整建议字段
每条记录需写入 `difficulty_adjustment_suggestion`：

- `up`：本次表现支持上调
- `keep`：本次表现支持保持
- `down`：本次表现支持下调

该字段仅代表单次建议；最终难度以 dashboard 的两次趋势规则决定。

## 5. 用户自述型评估补充规则
- 当用户提供的是“题册作答结果 + 错因说明”而非完整解题过程时，允许进行“用户自述型评估”。
- 用户自述型评估仍需写入统一记录结构，但应增加 `assessment_basis` 字段，取值为 `full_solution` 或 `self_report`。
- 在 `self_report` 场景下：
  - `correctness` 主要依据用户自述的对错情况；
  - `thinking_quality` 重点依据用户对错因的识别深度、方法意识与是否能复盘错误根源；
  - `missing_steps` 可记录“缺少完整推导，无法核验的关键环节”。
- 若用户明确说明“题目最终做对，但耗时长/不熟练/看答案后才会”，可降低 `thinking_quality`，但不必直接判为错误。

## 6. 错题 ABC 分级规则（复习索引）
- A级：纯计算错误（如代数化简失误、求导漏项、符号错误、分母乘错等）。
- B级：可下笔、有思路但过程不完整或方法选择不稳（能尝试但未完整做出）。
- C级：无法下笔或基本无解题入口（如“完全不知道怎么做”“未作答”）。
- 该分级用于“错题复习索引”文件，优先记录题号，不替代原始评分字段。
